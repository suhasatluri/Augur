Failure Modes

  • yfinance returns None for some fields: Each bias score component falls back to 0.5 independently. Simulation continues with reduced differentiation.
  • yfinance completely fails (network/rate limit): Entire structured data block empty. Bias score defaults to 0.50. Seeds rely on Claude general knowledge only. Quality drops from ~0.93 to ~0.80.
  • Yahoo Finance blocks requests: Known risk — Yahoo has historically blocked high-volume scrapers. V1 mitigation: slow layer caches for 7 days, so each ticker hits yfinance once per week maximum.

Mitigations

V1 — IMPLEMENTED Cache yfinance data aggressively in slow layer (7-day TTL). One fetch per ticker per week under normal usage.

V2 — PLANNED Add Alpha Vantage as secondary data source. If yfinance fails, attempt Alpha Vantage before falling back to Claude-only mode. Requires free API key (5 calls/minute, 500/day).

V2 — PLANNED Pin yfinance==0.2.x in requirements.txt to avoid breaking changes from upstream.

Default dev key augur-dev-key activates only when AUGUR_API_KEYS is unset — never in production.

Input Validation

  • Ticker: Validated against hardcoded ASX 200 whitelist (asx200.py). Rejects non-ASX tickers with 422.
  • Date: Parsed via date.fromisoformat() server-side. Rejects malformed dates with 422.
  • Rate limit: 10 simulations per API key per hour. In-memory counter (V1). Upstash Redis in V2.
  • Auth: X-API-Key header required on all endpoints except /health and /activity.

AFSL Boundary LEGAL

Augur stores: ticker, reporting_date, simulation_id, verdict, probability distributions, swing factors, round results.

Augur NEVER stores: user identity, portfolio data, transaction data, account balances, personal information, IP addresses.

All outputs carry mandatory disclaimer: "This is not financial advice. Research tool only. Augur does not hold an AFSL."

Disclaimer is hardcoded in prediction_synthesiser/models.py (DISCLAIMER constant), rendered in every API response containing predictions, and displayed in the frontend footer and results page.

Stale Job Cleanup IMPLEMENTED

On every Railway restart, augur_api.py runs _cleanup_stale_simulations():

  • Queries Neon for simulations with status running, negotiating, or forging created more than 10 minutes ago.
  • Marks them as failed with completed_at = NOW().
  • User sees "Simulation failed" and can resubmit.

V2 Improvements (Planned)

  • Upstash Redis job queue with automatic retry (3 attempts).
  • Checkpointing: if negotiation crashes after round 2, resume from round 3 instead of restarting.
  • Webhook notification to user on completion/failure.

V1.0 → V1.1 Cost Changes

  • Rounds reduced from 5 to 3: saves ~$0.20 per simulation (10 fewer Sonnet calls).

Retention Policy

V1: Indefinite retention of simulation data. No user data exists to protect. All stored data is derived from public market information (ASX tickers, publicly available financial data).

V2 (planned): Auto-purge simulation data older than 90 days to manage database size.
